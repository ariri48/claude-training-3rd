이론 1

왜 Claude Code인가?

채팅창이 아니라, 내 컴퓨터에서 직접 일하는 AI

채팅형 AI vs CLI형 AI

이론 1

같은 AI, 어디서 쓰느냐에 따라 완전히 다릅니다

💬 채팅형 AI (ChatGPT, Claude.ai)
브라우저 대화창. 물어보면 글로 답해 줍니다.
  • • 파일은 매번 업로드·복사·붙여넣기
  • • 대화창이 바뀌면 맥락이 사라짐
  • • 내 컴퓨터의 파일을 직접 못 만짐
  • • 여러 파일을 한 번에 다루기 번거로움
⌨️ CLI형 AI (Claude Code)
내 컴퓨터 안에서 파일을 직접 읽고 만듭니다.
  • • "이 폴더 엑셀 정리해줘" → 바로 실행
  • • 프로젝트 맥락을 계속 기억 (CLAUDE.md)
  • • Dooray 같은 업무툴과 연동 가능
  • • 폴더째 여러 파일을 한 번에 처리
🙋 이럴 땐 채팅형
가벼운 질문·아이디어·글 다듬기
🛠️ 이럴 땐 CLI형
내 파일·데이터를 실제로 처리·저장
🤝 둘 다 병행 OK
경쟁이 아니라 용도가 다른 도구
채팅형 = 말로 조언해주는 컨설턴트 · CLI형 = 내 옆에서 직접 일하는 비서
이론 1·5

Claude, 쓰는 창은 3가지 — 웹 · 앱 · CLI

같은 Claude라도 어디서 여느냐에 따라 할 수 있는 일이 다릅니다. 오늘 배우는 건 맨 오른쪽 CLI(Claude Code)예요.

구분🌐 웹 (claude.ai)💻 데스크톱·모바일 앱⌨️ CLI (Claude Code)
실행 위치브라우저설치형 앱터미널(검은 창)
주 용도대화·질문·글쓰기대화 + 자료 첨부내 파일·폴더를 직접 작업
내 컴퓨터 파일❌ 업로드만 가능❌ 업로드만 가능✅ 직접 읽고 만들고 저장
업무툴 연동(MCP)제한적일부✅ Dooray 등 풀 연동
요금제무료 플랜 있음무료 플랜 있음⚠️ 유료 전용 (Pro/Team/Max)
🔑 Claude Code(CLI)는 무료 플랜으로 로그인되지 않습니다. Pro·Team·Max 중 하나의 유료 구독이 있어야 실행돼요. (웹·앱은 무료로도 맛보기 가능)

2026년 나온 Claude Cowork는 터미널 없이 쓰는 AI 비서예요. 오늘 배우는 Claude Code와 용도가 다릅니다.

구분⌨️ Claude Code (오늘)🖥️ Claude Cowork

속도·비용텍스트 처리라 빠르고 저렴스크린샷 분석이라 느리고 토큰 많이 씀
내 컴퓨터·업무툴전체 접근 · Dooray 풀 연동접근이 제한적

파일 형식 기초

자주 보게 될 파일 확장자 5종

파일 이름 끝에 붙는 .html·.md 같은 건 "이 파일은 이런 용도"라는 표시예요. 외울 필요 없습니다.

확장자역할어디서 열리나요?언제 보게 되나요?
.html웹 페이지크롬 같은 브라우저대시보드·디자인 메일 만들기
.md마크다운 문서메모장·VS Code보고서, CLAUDE.md 규칙 파일, 스킬 파일
.xlsx엑셀 파일엑셀분석 대상 데이터, 명단
.py파이썬 스크립트터미널에서 실행자동화 스크립트
.env비밀 값 보관메모장(숨김 파일)토큰·API 키 저장 (예: Dooray 연동)
💡 Claude Code가 만들어주는 파일은 대부분 위 형식 중 하나예요. 더블클릭하면 알맞은 앱이 자동으로 열립니다.

실습 1

설치하기 — 명령어 한 줄이면 끝

천천히, 한 줄씩 같이 칩니다

보조강사가 순회하며 1:1 지원

STEP 1 · 터미널 열기

"터미널"은 컴퓨터에 말 거는 창입니다

🪟 Windows
시작 버튼 → PowerShell 검색 → 실행
파란/검은 창이 뜨면 정상
🍎 Mac
Spotlight(⌘+Space) → 터미널 입력 → 실행
흰/검은 창이 뜨면 정상
✅ 안 망가집니다
명령을 잘못 쳐도 "그런 명령 없음"이라고 뜰 뿐
📋 복붙이 기본
직접 타이핑 X — 가이드 명령을 복사·붙여넣기
↩️ Enter로 실행
붙여넣고 Enter 를 눌러야 실행됨
🔤 창에 보이는 기호, 겁먹지 마세요
$ 또는 % = "명령 기다리는 중" 표시(입력 X) · ~ = 내 홈 폴더 · 붙여넣을 때 $ 뒤 내용만 복사
검은 창이 떠도 놀라지 마세요. 아무거나 쳐도 컴퓨터는 안 망가집니다. ✋
STEP 2 · 명령어 한 줄 붙여넣기

설치는 명령어 한 줄 — 복사해서 붙여넣기

📖 실습자료 1-A

예전엔 Node.js·npm을 따로 깔았지만, 이제는 Claude Code가 알아서 다 설치합니다. 아래에서 내 OS 줄만 복사하세요. (⚠️ 옛날 npm install 방식은 이제 안 씁니다)

🍎 Mac / Linux
# 터미널에 붙여넣고 Enter $ curl -fsSL https://claude.ai/install.sh | bash
Homebrew를 쓴다면 → brew install --cask claude-code
🪟 Windows (PowerShell)
# PowerShell에 붙여넣고 Enter $ irm https://claude.ai/install.ps1 | iex
로컬 작업엔 Git for Windows가 있어야 해요 (git-scm.com)
✅ 설치 확인
$ claude --version 2.1.x (Claude Code) # 버전 번호가 뜨면 성공
설치는 오늘 한 번만. 다음부터는 claude 한 단어면 켜집니다.
STEP 3 · 로그인 & 첫 대화

실행 → 브라우저 로그인 → 첫 질문

📖 실습자료 1-B, 1-C
# 아무 폴더에서나 실행 $ claude # 브라우저가 자동으로 열립니다 → Claude 유료 계정으로 로그인 # (Pro / Team / Max 중 하나 필요)
🗣️ 첫 대화 — 이렇게 던져보세요
> 지금 이 폴더에 어떤 파일이 있는지 한국어로 알려줘 > 오늘 날짜와 요일 알려줘
대화창이 뜨고 한국어로 답하면 설치 성공입니다. 🎉 (손 들어서 조교 확인받기)
에러가 나도 당황 금지

자주 나는 문제 · 바로 이렇게

📖 실습자료 1-H
❓ command not found: claude
터미널을 완전히 닫고 다시 열기. 그래도 안 되면 설치 명령 재실행.
❓ Windows에서 실행 정책 오류
PowerShell을 관리자 권한으로 다시 열거나, WSL(Ubuntu)에서 설치.
❓ 설치 중 빨간 글씨가 잔뜩
대부분 경고라 무시 OK. 마지막에 claude --version 이 되면 성공.
❓ 로그인 브라우저가 안 열림
터미널에 뜬 URL을 직접 복사해서 브라우저에 붙여넣기.
❓ 권한 오류(EACCES) — Mac
가이드의 권한 명령 그대로 복사. 모르면 조교 호출.
🆘 그래도 안 되면
손 들기 → 보조강사 1:1. 혼자 오래 붙들지 마세요.
실습 2

Skills + Dooray MCP 연동

Claude Code를 "풀옵션"으로, 그리고 Dooray와 연결

보조강사 순회 1:1 지원

STEP 4 · Skill 이해 → 설치

Skill = "일하는 방법"을 미리 담은 확장팩

📖 실습자료 1-I

Claude는 원래도 똑똑하지만, Skill은 "이런 일은 이 순서·이 형식으로 해"를 정해둔 묶음입니다. 그래서 결과가 매번 일정하고, 한마디만 해도 알아서 처리합니다 (예: oh-my-skills).

📊
문서·데이터 처리
엑셀·PDF·CSV 정리 스킬
🧾
보고서 템플릿
주간보고·회의록 형식 자동 적용
🔌
업무툴 연동 묶음
Dooray 등과 함께 쓰는 작업 세트
# 가이드에 적힌 설치 명령을 그대로 복사 → 실행 # 설치 후 Claude Code 안에서 확인 > 사용 가능한 skill 목록 보여줘 > 엑셀 정리 skill 써서 이 파일 월별로 합쳐줘
✅ 오늘의 목표
"skill 목록이 뜬다" = 설치 성공. 활용은 기초 과정에서.
💡 앱 스토어처럼
필요한 기능만 골라 설치·삭제. 안 쓰면 지워도 됨.
Skill은 "기능 추가"일 뿐. 지금 다 이해 못 해도 됩니다 — 설치되어 있으면 성공.
개념 · MCP

MCP = Claude와 외부 도구를 잇는 표준 케이블

Model Context Protocol — 어떤 도구든 같은 방식으로 연결하는 "USB-C 규격" 같은 것.

🔌 MCP가 없으면
  • • Dooray 내용을 내가 복사해서 붙여넣어야 함
  • • 최신 상태를 매번 다시 가져와야 함
  • • 결과를 다시 Dooray에 손으로 옮겨 입력
🔗 MCP가 있으면
  • • Claude가 Dooray를 직접 조회해 태스크·프로젝트를 읽음
  • • 항상 실시간 최신 데이터로 작업
  • • "조회 → 정리 → 저장"이 한 번에 이어짐
📋 Dooray MCP
태스크·프로젝트·위키 조회 (오늘 연동)
📁 파일시스템
내 폴더의 문서를 직접 읽고 쓰기 (기본 내장)
🌐 그 외 수백 종
Slack·GitHub·DB 등 — 필요할 때 붙이면 됨
핵심: MCP는 Claude에게 "손과 눈"을 달아주는 것. 글로 조언하던 AI가 실제 도구를 직접 다루게 됩니다.
STEP 5 · Dooray MCP 토큰 → .env → 연동

Dooray를 터미널에서 조회하기

📖 실습자료 1-G
① 토큰 발급
Dooray → 설정 → API/개인 인증 토큰 발급 (가이드의 화면 순서대로)
② .env 에 저장
# .env 파일에 붙여넣기 DOORAY_TOKEN=발급받은_토큰
❓ .env 파일이 뭐예요?
토큰·비밀번호 같은 민감한 값을 따로 담아두는 메모장 파일이에요. 코드에 직접 적지 않고 .env 에만 넣어두면, 실수로 남에게 공유되는 걸 막을 수 있어요. (env = environment, 환경설정)
📁 어디에 저장?
claude를 실행하는 그 폴더(작업 폴더) 맨 위에, 파일 이름을 딱 .env 로 (앞에 점 하나, 확장자 없음)
✍️ 어떻게 작성?
한 줄에 이름=값 형식. 예: DOORAY_TOKEN=abc123 · 값에 따옴표·띄어쓰기 불필요
🛠️ 만드는 법
claude에게 ".env 파일 만들고 DOORAY_TOKEN 넣어줘" 라고 말하거나, 메모장으로 만들어 저장
💡 파일 이름이 점(.)으로 시작하면 평소엔 숨겨져 안 보일 수 있어요 — 정상입니다.
③ 연동 확인
> Dooray에서 내가 속한 프로젝트 하나만 알려줘 # 이름이 하나라도 나오면 연동 성공 🎉
⚠️ 토큰은 비밀번호입니다. 화면 공유·채팅에 노출 금지, .env 밖으로 복사 금지.
STEP 6 · 가벼운 실습

진짜 되는지 — 30초 만에 직접 써보기

📖 실습자료 3-F

설치·연동이 끝났으면, 실제로 한 번씩 굴려봐야 내 것이 됩니다. 아래를 순서대로 던져보세요.

🧩 실습 A · Skill 써보기
# 1) 설치 확인 > 사용 가능한 skill 목록 보여줘 # 2) 하나 골라 실제로 시켜보기 > 이 폴더 내용을 표로 정리하는 skill 있으면 써서 정리해줘
✅ 확인점: 목록이 뜨고, 스킬 이름이 답변에 언급되면 성공
🔗 실습 B · Dooray MCP 써보기
# 1) 연결 확인 (가볍게) > Dooray에서 내가 속한 프로젝트 하나만 알려줘 # 2) 한 걸음 더 — 직접 써보기 > 내 개인 프로젝트에 "MCP 테스트" 업무 글 하나 작성해줘
✅ 확인점: Dooray에 실제로 글이 하나 생기면 성공
💬 이렇게 쓰는 겁니다
"조회해줘 → 정리해줘 → 저장해줘"를 말로 이어서
🔁 안 되면
터미널 재시작 → 그래도 안 되면 손 들기 (조교 1:1)
🎯 여기까지 됐다면
기초 과정 준비 완료 — 실전은 다음 시간에
"설치했다"보다 중요한 건 "한 번 써봤다". 오늘 이 두 줄만 성공하면 충분합니다. 🙌

⌨️ 알아두면 편한 키
  • • Esc — 실행 중인 답변 중단
  • • ↑ — 직전에 보낸 프롬프트 다시 불러오기
  • • Shift+Enter — 줄바꿈(여러 줄 프롬프트)
  • • 파일 경로는 @ 로 자동완성
🗣️ 명령어 대신, 이렇게 말하면 됩니다
> 이 폴더 파일 목록 보여줘 > report.xlsx 열어서 요약해줘 > 방금 만든 파일 이름 바꿔줘
명령어는 "도구 조작", 실제 일은 한국어 문장으로. 슬래시(/)는 도구용, 그냥 글자는 업무용이라고 기억하세요.
