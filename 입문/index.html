이론 1

왜 Claude Code인가?

채팅창이 아니라, 내 컴퓨터에서 직접 일하는 AI

00:05–00:15 · 채팅형 AI vs CLI형 AI

이론 1

같은 AI, 어디서 쓰느냐에 따라 완전히 다릅니다

💬 채팅형 AI (ChatGPT, Claude.ai)
브라우저 대화창. 물어보면 글로 답해 줍니다.
  • • 파일은 매번 업로드·복사·붙여넣기
  • • 대화창이 바뀌면 맥락이 사라짐
  • • 내 컴퓨터의 파일을 직접 못 만짐
⌨️ CLI형 AI (Claude Code)
내 컴퓨터 안에서 파일을 직접 읽고 만듭니다.
  • • "이 폴더 엑셀 정리해줘" → 바로 실행
  • • 프로젝트 맥락을 계속 기억
  • • Dooray 같은 업무툴과 연동 가능
채팅형 = 말로 조언해주는 컨설턴트 · CLI형 = 내 옆에서 직접 일하는 비서
용어 정리 · 겁먹지 않기

오늘 나오는 낯선 단어, 딱 이 정도만

🖥️ CLI / 터미널
마우스 대신 키보드로 컴퓨터에 명령하는 검은 창. 무서운 게 아니라 "말 거는 창".
📦 Node.js
Claude Code가 돌아가려면 필요한 엔진. 자동차의 엔진이라 생각하세요.
🌿 Git
파일 버전을 관리·내려받는 도구. 실습자료도 Git으로 받습니다.
🧩 Skill
Claude Code에 기능을 추가하는 확장팩. "풀옵션"으로 만들어 줍니다.
🔌 MCP
Model Context Protocol. Claude를 외부 도구(Dooray 등)에 연결하는 표준 케이블.
📝 CLAUDE.md
"내 업무는 이렇습니다"를 적어두는 나만의 설명서. (기초 과정에서 작성)

실습 1

설치하기 — Node · Git · Claude Code

천천히, 한 줄씩 같이 칩니다

00:25–01:10 · 보조강사가 순회하며 1:1 지원

STEP 0 · 터미널 열기

"터미널"은 컴퓨터에 말 거는 창입니다

🪟 Windows
시작 버튼 → PowerShell 검색 → 실행
🍎 Mac
Spotlight(⌘+Space) → 터미널 입력 → 실행
검은 창이 떠도 놀라지 마세요. 아무거나 쳐도 컴퓨터는 안 망가집니다. ✋
STEP 1~3 · 엔진 확인 → 도구 설치

한 줄씩 복사해서 붙여넣기

① Node.js = 엔진
# 버전이 뜨면 이미 설치됨 $ node --version v22.15.0 # 안 뜨면 → nodejs.org 에서 설치
② Git = 파일 관리·다운로드
$ git --version git version 2.45.0 # 안 뜨면 → git-scm.com 에서 설치
③ Claude Code = 자동차
# 1~2분 소요, 빨간 글씨는 대부분 무시해도 됨 $ npm install -g @anthropic-ai/claude-code added 312 packages # = 성공
설치는 오늘 한 번만. 다음부터는 claude 한 단어면 켜집니다.
STEP 4~5 · 로그인 & 첫 대화

실행 → 브라우저 로그인 → 첫 질문

# 아무 폴더에서나 실행 $ claude # 브라우저가 자동으로 열립니다 → Claude 유료 계정으로 로그인 # (Pro / Team / Max 중 하나 필요)
🗣️ 첫 대화 — 이렇게 던져보세요
> 지금 이 폴더에 어떤 파일이 있는지 한국어로 알려줘 > 오늘 날짜와 요일 알려줘
대화창이 뜨고 한국어로 답하면 설치 성공입니다. 🎉 (손 들어서 조교 확인받기)
에러가 나도 당황 금지

자주 나는 문제 · 바로 이렇게

❓ command not found: claude
터미널을 완전히 닫고 다시 열기. 그래도면 설치 명령 재실행.
❓ node/npm 이 없다고 나옴
nodejs.org LTS 설치 후 터미널 재시작.
❓ 빨간 경고(warn)가 잔뜩
경고(warning)는 대부분 무시 OK. 마지막에 added…가 뜨면 성공.
❓ 로그인 브라우저가 안 열림
터미널에 뜬 URL을 직접 복사해서 브라우저에 붙여넣기.
❓ 권한 오류(EACCES) — Mac
가이드의 권한 명령 그대로 복사. 모르면 조교 호출.
🆘 그래도 안 되면
손 들기 → 보조강사 1:1. 혼자 오래 붙들지 마세요.
실습 2

Skills + Dooray MCP 연동

Claude Code를 "풀옵션"으로, 그리고 Dooray와 연결

01:10–01:50 · 보조강사 순회 1:1 지원

STEP 6 · Skills 설치

Skill = Claude Code 확장팩

자주 쓰는 기능 묶음을 한 번에 추가합니다 (예: oh-my-skills).

# 가이드에 적힌 설치 명령을 그대로 복사 → 실행 # 설치 후 Claude Code 안에서 확인 > 사용 가능한 skill 목록 보여줘
Skill은 "기능 추가"일 뿐. 지금 다 이해 못 해도 됩니다 — 설치되어 있으면 성공.
개념 · MCP

MCP = Claude와 외부 도구를 잇는 표준 케이블

🔌 MCP가 없으면
Dooray 내용을 내가 복사해서 Claude에 붙여넣어야 함
🔗 MCP가 있으면
Claude가 Dooray를 직접 조회해서 태스크·프로젝트를 읽어옴
Model Context Protocol — 어떤 도구든 같은 방식으로 연결하는 "USB 규격" 같은 것.
STEP 7 · Dooray MCP 토큰 → .env → 연동

Dooray를 터미널에서 조회하기

① 토큰 발급
Dooray → 설정 → API/개인 인증 토큰 발급 (가이드의 화면 순서대로)
② .env 에 저장
# .env 파일에 붙여넣기 DOORAY_TOKEN=발급받은_토큰
③ 연동 확인
> Dooray에서 내 프로젝트 목록 보여줘 # 목록이 나오면 연동 성공 🎉
⚠️ 토큰은 비밀번호입니다. 화면 공유·채팅에 노출 금지, .env 밖으로 복사 금지.
기본 명령어 3개만

이것만 알면 오늘은 충분합니다

/help
쓸 수 있는 명령 목록 보기
/clear
대화 맥락 초기화 (새 작업 시작할 때)
/exit
Claude Code 종료
나머지는 그냥 한국어로 말하면 됩니다. 명령어를 외울 필요 없어요.
